Pedro Cruz aka gankd

Penetration Tester at Hakai Security

HTML-to-PDF conversion libraries power critical business workflows—generating invoices, reports, contracts, and certificates at massive scale. Yet despite widespread adoption, these libraries remain understudied for security vulnerabilities. This talk explores how unsanitized user input reflected into PDF-generating HTML unlocks critical vulnerabilities across six widely-used open-source engines: WeasyPrint, ReportLab, BrowserShot, Dompdf, ReLaXed, and mPDF. Drawing from pentesting engagements, the speaker demonstrates exploitation of 0-days including Local File Read, SSRF via open redirect bypass, Remote Code Execution through insecure deserialization, template injection, Local File Inclusion, file existence oracles, and Denial of Service attacks. Attendees will learn to identify vulnerable patterns during code review and penetration testing, understand how HTML injection becomes a gateway to server-side attacks, and implement mitigations when deploying these libraries in production.

Arad Donenfeld

Attacks and Exploits Developer

The Agents of Chaos: AI Driven Malware Generation

As AI agents catch wind across offensive security, from social engineering to vulnerability research, it was only a matter of time before malware followed. While much of the conversation focuses on targeting AI, generating malicious payloads at runtime, or “vibe coding” malware, we explored a different direction: an agent where AI is the sole participant in the malware creation process. We begin by examining how this idea emerged and compare different models, evaluating code quality, prompt effectiveness, and agent design. We then dive into the generation process itself, covering the limitations of earlier approaches, how we addressed them, and how to build workflows that maximize capability, randomization, signature evasion, and large-scale sample generation. Finally, we explore how similar methods are already appearing in the wild and discuss what they mean for both attackers and defenders.

Arnav Tripathy

Cloud Security Engineer

Building a digital beehive: The cluster that wasn't real, but the attacks were

Kubernetes is everywhere now—and so are attacks against it. Misconfigurations, exposed APIs, and overly permissive RBAC have made clusters a prime target, while most security tools stop at scanning images and YAML instead of showing what attackers do after they get in. This talk introduces KubeDecoy – a lightweight, open-source Kubernetes honeypot built from familiar components: vcluster, Falco, Falcosidekick, and NGINX. The goal is simple: stand up a convincing fake cluster, expose realistic attack surfaces, and quietly observe how real adversaries interact with them. We’ll walk through the architecture, a live-style deployment on Minikube, and practical workarounds for vcluster’s limitations using only open-source components. Along the way, we’ll map KubeDecoy’s detections to the Kubernetes Threat Matrix, highlight where it shines (and where it doesn’t), and show how you can apply these ideas in your own clusters—whether you’re blue team, red team, or somewhere in between.

When regulations collide : A crisis management story

David Papp

Gen0Sec, CEO

Detect, Block, Jail: Kernel-Native Defense in Depth with eBPF

Traditional network and endpoint security tools live in userspace, paying a latency tax at every inspection point. This talk shows how to move detection and enforcement into the kernel itself using two open-source eBPF projects: Synapse, an Active NDR engine doing inline blocking with JA4+ fingerprinting and zero TLS decryption, and Jailer, an eBPF-based mandatory access control system. Together they form a kernel-resident defense-in-depth stack — with live demos of real attacks stopped before the syscall returns.

Trendyol, Application Security Engineer

Your AI Just Leaked a Secret

In this talk, I present a three-month analysis of recent GitHub activity across thousands of public repositories. Rather than scanning source code alone, I used AI to analyze commit messages for signals of secret exposure, such as references to credential removal, leaked keys, or emergency fixes. These messages act as a side channel, often revealing that sensitive data was committed shortly before being removed. The analysis shows a recurring pattern where secrets originate from AI-assisted development workflows, including boilerplate code and configuration generated during fast-paced vibe coding sessions. This methodology led to the discovery and responsible disclosure of real-world issues, resulting in Hall of Fame acknowledgements from organizations such as Google, Microsoft, and Red Hat.

Don’t Trip the Breaker: A Practitioner's Guide to Safe Power Grid Pentesting

Pentesting critical infrastructure like a power grid is a high-stakes game where "move fast and break things" is a recipe for disaster. This session provides a transparent look on a regional grid, breaking down a safety-first methodology that balances deep technical discovery with operational uptime. We start with the Rules of Engagement (RoE) workshop, learning to map environments using digital twins and mirrored test benches. From there, we leverage passive traffic analysis and protocol-specific inspection to identify firmware CVEs and insecure "clear-text" command paths without touching a live PLC. The core demonstrates mapping the effectiveness of the Purdue Model. We’ll walk through lateral movement scenarios—testing if corporate Wi-Fi can jump the DMZ into Levels 3-0—and identify "low-hanging fruit" like default credentials. You’ll leave with a blueprint for OT assessments that satisfy both security auditors and plant engineers without risking a blackout.

Andrew Schwartz

Principal Detection Engineer @ Huntress

From Code to Sigma: A Detection Engineer's Journey Through the LDAP Wilderness

Active Directory reconnaissance tools like BloodHound, Impacket, and SOAPHound are the attacker's first move in enterprise compromises, yet detecting their LDAP queries remains one of the hardest problems in sec-ops. This talk chronicles a six month journey from writing my first broken Sigma rule to building a complete, evasion resistant LDAP detection stack. You'll learn why traditional signature based detection fails spectacularly, how to think like both an attacker and a parser, and how mathematical approaches can outsmart evasion techniques. We'll cover OID transformations that break your rules, whitespace variations that mock your regex, hidden LDAP parameters that bypass your detections, and ultimately, statistical methods that make evasion mathematically impossible. This isn't theory. Techniques are battled tested in production environments with working Sigma rules, real attack logs, and actual false positive rates. Leave with knowledge you can deploy Monday morning.

Medical Device Kill Chain: From Debug Port to Patient Impact

The hospital room looks secure. The patient monitor beeps steadily. The infusion pump delivers medication at precisely the programmed rate. The imaging system uploads its results to the cloud. Everything is working exactly as designed.Except the firmware on that patient monitor contains a hardcoded SSH private key. The infusion pump has an exposed JTAG interface that anyone with a $30 adapter can connect to. And none of this has been patched, because nobody has formally mapped how these individual weaknesses chain together into a complete attack. That is the problem the Medical Device Kill Chain was built to solve.

Beyond SHA Pinning: Security for CI/CD Pipelines

Liam Follin

Principal Security Researcher

Even more alert(1)

Cross-site scripting is often treated as a solved problem. Output is escaped, frameworks are opinionated, and many assume that without obvious use of innerHTML or eval, XSS is no longer possible. That assumption is exactly why modern XSS still exists. These bugs live almost entirely in the browser, emerging from JS behaviour and DOM sinks. XSS is a client-side problem rather than a server-side issue. Many of the vulnerabilities shown never touch an HTTP response. Input stays in the client, and becomes executable because of how the browser interprets it. We then practice DOM-based exploitation using the DOMinic labs (github.com/gr4y-r0se/DOMinic), focusing on runtime DOM behaviour. Lastly, we move on to JS reflection using the ReflectJS labs (github.com/gr4y-r0se/ReflectJS), showing how constrained input inside script contexts can still reach execution. We close by connecting exploitation to detection and defence, with an emphasis on understanding JavaScript as it actually runs.

Lidor Ben Shitrit aka Lidor B. (thisis0xczar)

Founding Team Vulnerability Researcher @ Novee Security

Pre-auth RCE in Enterprise Java: When Middleware Becomes the Exploit

Enterprise Java platforms still expose critical pre-authentication attack paths through middleware features that were never designed to handle untrusted input. In this talk, we present multiple real-world pre-auth remote code execution chains discovered in widely deployed enterprise Java platforms. We show how attackers can reach internal-only execution surfaces through routing logic, dispatcher behavior, authentication glue code, and unsafe object construction paths. Our flagship case study demonstrates how an unauthenticated attacker escapes an exposed API surface, reaches privileged internal endpoints, and escalates into unsafe deserialization through multiple independent sinks, including permissive XStream configuration and a secondary ObjectInputStream path lacking JEP-290 filtering. We also present a second pre-auth RCE chain where token trust failures and template expansion logic combine into direct code execution through Groovy evaluation.

Discovering 0-Days in HTML-to-PDF Generators (40 Million Monthly Targets)

HTML-to-PDF conversion libraries power business workflows but remain understudied for security vulnerabilities. This talk explores how unsanitized user input in PDF-generating HTML unlocks critical vulnerabilities across six engines: WeasyPrint, ReportLab, BrowserShot, Dompdf, ReLaXed, and mPDF. Drawing from pentesting, the speaker demonstrates 0-day exploits including Local File Read, SSRF, Remote Code Execution, template injection, Local File Inclusion, file existence oracles, and Denial of Service attacks. Attendees will learn to identify vulnerable patterns during code review, understand how HTML injection enables server-side attacks, and implement mitigations in production.
